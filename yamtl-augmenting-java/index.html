Yamtl Augmenting Java

15 Jan 2020 - -

YAMTL: Augmenting Java Programs with Model Transformation

Model transformation is traditionally used to translate models between different modeling languages and it can also be used to manipulate data that is modeled using an object-oriented notation, such as UML. This defining characteristic makes it an ideal candidate for developing low-code application platforms by, for example, automating both translation tasks and reverse engineering tasks.

YAMTL is a model transformation (MT) language that is defined as an internal domain-specific language of Xtend and, thereby, reuses advanced IDE support and out-of-the-box interoperability with Java, including Android. Its transformation engine is one of the fastest thanks to working with compiled transformations.

YAMTL is a model transformation language fully interoperable with Java, expressive and very fast - arturboronat.info/yamtl

YAMTL comes equipped with advanced model transformation features that can help in implementing any kind of model transformation: as expressive as state-of-the-art model transformation languages, such as ATL; powerful model pattern matching with local-search programmable plans; reuse via multiple rule inheritance and module composition; compiled to Java; full interoperability from/to Java; additional facilities inherited from Xtend (templates for code generation, IDE, debugger, and Java extensions).

In a recent survey on the future of MT languages, respondents enumerated a number of reasons for not using MT languages. In the table below, we indicate how YAMTL tackles some of those challenges:

Reason for not using MT YAMTL solution
Complex setup and IDE-specific dependencies Use mature Xtend IDE/Java ecosystem
No real debugging Use mature Xtend/Java debugging techniques
Not adapted to the web As a Maven dependency, YAMTL can be used in Java web projects (server-side)
Performance Our experiments [MODELS18, TTC18, FASE19, TTC19] show that performance is a strong point of YAMTL
ATL when simple transformations, Java when complex transformations YAMTL raises the level of abstraction for complex transformations
Cost in training staff Use plain Java with the YAMTL dependency

Trying YAMTL…

Using a YAMTL transformation is as easy as…

  1. Importing the YAMTL dependency in your Java project.
  2. Declaring a model transformation. See any of the following examples:
    • CPS
    • CD2RDBMS
    • TTC18
    • TTC19a
    • TTC19b
  3. Executing the model transformation:
// INSTANTIATE THE TRANSFORMATION DECLARATION
val xform = new cd2db

// PREPARE MODELS
xform.loadInputModels(#{'cd' -> inputModelPath})

// EXECUTE TRAFO 
xform.execute()

// STORE MODELS
var String outputModelPath = '''src/main/java/cd2db/targetInitial.xmi'''
xform.saveOutputModels(#{'db' -> outputModelPath})
